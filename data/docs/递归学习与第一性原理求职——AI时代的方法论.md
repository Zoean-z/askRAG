> 来源：B站 BV16MAnzEEpJ — Gabriy（23岁辍学，OpenAI研究科学家，参与Sora，年薪$700K）

## 一、递归学习法

### 模型

传统学习是**广度优先遍历**：从基础层逐层向上，直到触达目标。
递归学习是**深度优先搜索**：从目标节点出发，沿依赖链向下按需展开。

```
传统路径：  数学 → 编程 → ML → DL → 视频生成    （广度优先，耗时数年）
递归路径：  视频生成 → 扩散模型 → 去噪原理 → 概率分布 → 需要时再补数学
```

AI（GPT/Claude）在此过程中充当**按需知识图谱遍历器**——你指定当前节点，它返回依赖节点及解释。

### 操作协议

1. 向AI要求目标领域的最小可运行实现
2. 逐行/逐概念追问，遇到不懂的概念立刻递归展开
3. 追问直到触达你已有知识的锚点，然后回溯
4. 重复，日均追问量级：百次

### 适用条件与边界

| 条件 | 有效 | 失效 |
|------|------|------|
| 领域可分解性 | 软件、AI、数据——组件化、可拆解 | 外科手术、飞行——身体技能无法递归追问获得 |
| 基础知识依赖 | 概念可按需补——数学公式查到即用 | 强序列依赖——高等数学证明需要前置定理链 |
| 验证可得性 | 代码可运行验证、结果可观测 | 无法快速验证的领域（医学诊断、法律推理） |
| AI知识覆盖 | 主流技术栈——AI训练数据充分 | 前沿/冷门/保密领域——AI会hallucinate |

**核心风险**：递归学习生成的是**工作知识**（能用）而非**结构知识**（能解释为什么）。长期只做递归学习会形成瑞士奶酪式知识结构——满是孔洞但表面看似完整。需要周期性做广度补全。

### 与库内文档的连接

- 本质上是 `跳出框架：认知增强的完整地图` 中维度C（行动-学习循环）+ 维度D（环境工具）的具体实例
- AI作为"24小时教师"是维度D中"工具环境"的极端情况
- 递归追问的有效性取决于提问者的**元认知能力**——知道自己不知道什么。这是认知增强文档中诊断的核心瓶颈

## 二、第一性原理求职法

### 模型

传统求职依赖**信号传递**（Spence, 1973）：学历、证书、简历关键词 → HR筛选 → 面试 → 录用。
第一性原理求职跳过信号层，直接提供**能力证明**。

```
传统路径：  信号（学历）→ 守门人（HR）→ 决策者     （多层衰减，假阳性高）
直接路径：  作品（能力证明）→ 决策者 → 试用验证    （一步到位，风险可控）
```

### 操作协议

**第一步：构建3秒可验证的能力载体**
- 用1周时间，借助AI做一个可运行的小产品（工具/网站/自动化脚本）
- 判断标准：陌生人打开后3秒内能得出"这人能做事"的结论
- 不是展示技术深度，是展示**交付能力**

**第二步：绕过守门人，直达决策者**
- 联系创始人/技术负责人/业务负责人
- 话术结构：展示作品 + 提出零风险试用（"我可以免费帮你们做一周"）

**第三步：免费试用 = 非对称下注**
- 你的成本：1周时间
- 你的收益（无论结果）：真实项目经验 + 可展示的案例
- 对方的成本：几乎为零
- 对方的收益：免费劳动力 + 低风险评估候选人

这是 `杠杆·结构·奇点` 中"非对称风险"概念的求职应用——小亏损、大收益、正期望值。

### 适用条件与边界

| 条件 | 有效 | 失效 |
|------|------|------|
| 岗位类型 | 技术/创意/builder类——产出可展示 | 管理/合规/制度性岗位——需要资质证书 |
| 公司阶段 | 创业期/成长期——决策者可触达、流程灵活 | 大厂/体制内——HR流程不可绕过 |
| 行业监管 | 弱监管行业（科技、内容、设计） | 强监管行业（医疗、法律、金融持牌岗） |
| 作品可做性 | 目标领域的MVP可在1周内独立完成 | 需要团队/数据/资质才能产出有意义成果 |

**核心风险**：幸存者偏差。Gabriy的案例被传播恰恰因为它是异常值。同样方法的大量失败案例不会被拍成视频。这套方法提高了成功概率，但不保证结果。

## 三、底层结构：两个方法的共同模型

两个方法共享同一个元策略：

> **跳过中间层，直接连接输入与输出，用快速迭代替代前置准备**

| 维度 | 传统路径 | 递归/第一性原理路径 |
|------|---------|-------------------|
| 学习 | 先学完基础再开始 | 先开始再按需补基础 |
| 求职 | 先积累信号再求职 | 先展示能力再补资质 |
| 时间分配 | 前置投入重，后期回报 | 全程轻量迭代，边做边验证 |
| 风险结构 | 高沉没成本（学了可能用不上） | 低沉没成本（每步都有即时反馈） |
| 依赖假设 | 知识/信号有稳定价值 | 能力可被直接观测和验证 |

这个元策略在AI时代被放大，因为AI降低了"按需获取知识"和"快速构建作品"的成本。但它的有效性取决于一个前提：**目标领域允许快速验证**。如果验证成本高、反馈周期长，这个策略退化为盲目试错。
